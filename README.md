# 对话系统中缓存利用与总结窗口的成本优化

> 本仓库为单页应用 [**对话成本策略比较器**](https://duohbshuijiao.github.io/SumOrNot/) 的理论说明与实现注记。网页中的折线、区间表与打平点均基于下述**简化计费模型**估算，用于把握趋势与数量级，**不等于**商业 API 的完整对账单。

---

## 摘要

大语言模型接口常对「与上一轮请求自开头起完全相同的前缀」按较低单价（缓存读）计价，对新增后缀按普通输入价计价。对话策略若始终堆积完整历史，则长前缀可反复命中缓存；若采用摘要与窗口裁剪，则前缀在相邻轮次间更易发生**结构性变化**，缓存收益与摘要所占用 token 之间存在权衡。本文档在与实现一致的假设下，给出**策略 A**（全量前缀缓存的基准）的闭式成本、**策略 B1/B2**（滚动窗口与批量裁剪）的逐轮成本递推，并说明摘要强度、压缩比与「策略 B 不劣于 A」的轮次区间在工具中的**数值**刻画方式。

---

## 1. 背景与计费模型

某大语言模型 API 按 token 数计费，单价以**元 / 百万 token** 记。记：

- $c_{\mathrm{in}}$：普通（非缓存）输入单价；
- $c_{\mathrm{cache}}$：缓存命中的输入段单价，通常 $c_{\mathrm{cache}} \le c_{\mathrm{in}}$；
- $c_{\mathrm{out}}$：输出 token 单价。

**前缀缓存规则（与页面假设一致）**

- 若当前请求与**上一轮**请求从首字符起有一段**完全相同的连续前缀**，则该段长度 $L_{\mathrm{common}}$ 上按 $c_{\mathrm{cache}}$ 计费，其余 $L - L_{\mathrm{common}}$ 按 $c_{\mathrm{in}}$ 计费。
- 若前缀在任意位置与上一轮不一致，则**整段输入**均按 $c_{\mathrm{in}}$ 计价（可视为 $L_{\mathrm{common}} = 0$ 的极端）。
- 首轮无历史，全部输入按 $c_{\mathrm{in}}$。

**单轮输入成本**（以 token 长度假设为连续量）：

$$
C_{\mathrm{in}}(L, L_{\mathrm{common}}) = \frac{1}{10^6} \left( L_{\mathrm{common}} \, c_{\mathrm{cache}} + (L - L_{\mathrm{common}}) \, c_{\mathrm{in}} \right) \quad (\text{元}).
$$

输出成本：每轮若输出约 $A$ token，则 $C_{\mathrm{out}} = 10^{-6} A \, c_{\mathrm{out}}$。

## 2. 对话系统设定与符号

| 符号 | 含义 |
|------|------|
| $P$ | 系统提示（system prompt）token 数，会话级常数 |
| $A$ | 每轮助手回复 token 数，取常数近似 |
| $N$ | 总对话轮数（横轴） |
| $K$ | 每 $K$ 轮产生/刷新一次对话摘要，摘要长度 $S$（或由内节压缩比确定） |
| $H$ | 窗口中最多保留的**原始**助手回复轮数 |
| $S$ | 摘要长度假定（token） |

**摘要与压缩比（页面中的两种模式）**

- 直接指定 $S$；
- 或设压缩比 $1:R$，并令 $S = \frac{K \cdot A}{R}$。

实现见 `index.html` 中 `summaryS` 的取值；下文统一记摘要贡献长度为 $S$（即 `summaryS`）。

## 3. 三种上下文管理策略

### 3.1 策略 A：全缓存上下文（基准）

不总结、不截窗。第 $t$ 轮（$t \ge 1$）请求侧输入为系统提示后接前 $t-1$ 轮全部助手回复：

$$
\text{输入}_t = P + (t-1)\,A .
$$

与第 $t-1$ 轮相比，**新增**仅为一轮长度 $A$ 的后缀，此前 $P + (t-2)A$ 为共同前缀。因此在理想情况下：

- 第 1 轮：长度为 $P$，$L_{\mathrm{common}}=0$；
- 第 $t\ge 2$ 轮：$L = P + (t-1)A$，$L_{\mathrm{common}} = P + (t-2)A$。

于是第 $t$ 轮**输入**侧 token 中，$A$ 按 $c_{\mathrm{in}}$，$P+(t-2)A$ 按 $c_{\mathrm{cache}}$。对 $N$ 轮**累计**（含输出）在「元」尺度下为（与 `costA` 闭式一致）：

$$
\mathcal{C}_A(N) = 10^{-6} \left[
P\, c_{\mathrm{in}}
+ (N-1) P\, c_{\mathrm{cache}}
+ (N-1) A\, c_{\mathrm{in}}
+ \frac{(N-1)(N-2)}{2}\, A\, c_{\mathrm{cache}}
+ N A\, c_{\mathrm{out}}
\right].
$$

该式是单轮求和
$\sum_{t=1}^{N} \left( C_{\mathrm{in}}^{(A)}(t) + C_{\mathrm{out}} \right)$ 的**闭式**，其中 $\frac{(N-1)(N-2)}{2} A$ 项来自等差数列 $\sum_{t=2}^{N} (t-2)A$ 的缓存段长度之和。

> **与「动态规划」的类比：** 若令状态为「累积助手 token 行数」与已付前缀，每轮在「上一层前缀长度」上增加 $A$ 新输入、扩大缓存段，则总输入成本对轮数的增长呈现二次项 $N^2$ 量级（与 $(N-1)(N-2)/2$ 一致），适合作为课堂 DP 的连续化极限。

### 3.2 策略 B1：持续滚动窗口总结

**文本结构**（$t$ 足够大、摘要已存在时）：

$$
\text{输入} \approx P + S + \min(t-1,\, H) \times A\ \text{（截断在窗口内，更早轮次由 }S\text{ 承载）}.
$$

**题目中的保守缺省假设：** 相邻轮因滑窗/摘要位势变化，前缀频繁「结构性」改变，**可近似认为 B1 全程无缓存优势**，整段输入按 $c_{\mathrm{in}}$ 计（$L_{\mathrm{common}}=0$），便于与 A 作解析比较。

**本工具中的实现：** 若选择「滚动窗口」模式，代码在 $n>H$ 等段采用**显式** $L$ 与 $L_{\mathrm{common}}$ 规则（`simulateB` 中 `bMode === 'rolling'`，含「满窗后」与摘要刷新轮的保守处理），**并不**完全等于「全程 $c_{\mathrm{in}}$」；界面「模型在算什么」中亦说明为**保守近似**。以页面曲线为准时，以代码为准；手算阈值时，请自行选用上段「全 $c_{\mathrm{in}}$」或下节与代码一致的逐轮量。

### 3.3 策略 B2：批量裁剪（周期性截断 + 可缓存增长段）

在两次**裁剪/刷新**点之间，输入长度往往**随轮次单调增加**，大段前缀与上一轮相同，**可出现** $L_{\mathrm{common}} \approx$ 上一轮的**全长** $L_{t-1}$，仅**新增**为「本轮追加的助手 token」或**摘要/裁剪**导致的前缀**重置**。

在摘要轮：摘要文本替换、前缀可能自 $P$ 起重置；在裁剪轮：丢弃一段早期原始回复、保留 $P+S$ 与近端窗口，$L_{\mathrm{common}}$ 在实现中取 `P`、`P+S` 或 `prevLen` 等分支（`simulateB` 中 `bMode === 'batch'` 与 `summaryRound` / `trimRound` 判定）。

**批量裁剪的阶段性输入长度**（与实现对应的一种分期描述）：

- 首次接入摘要后：$L \approx P + S + rA$，$r$ 为当前保留的原始轮数，随 $r$ 逐步增至 $H$ 直至裁剪；
- 若干轮内 $L$ 随 $r$ 增加，且 `common = prevLen`，输入侧新增近似为 $A \cdot c_{\mathrm{in}}$ 占主导；
- 在摘要/裁剪的特定轮次，`common` 下降，**普通输入**比例上升，形成周期性「锯齿」成本。

## 4. 问题要求与递推式

### 4.1 建立递推成本模型

**（1）策略 A**

令 $C_A(n)$ 为前 $n$ 轮**累计**成本（元）。有闭式 $\mathcal{C}_A(n)$ 于第3.1 节；亦可写递推

$$
C_A(0)=0, \qquad
C_A(n) = C_A(n-1) + \Delta_A(n),\quad n\ge 1,
$$

其中 $\Delta_A(n)$ 为第 $n$ 轮输入+输出，由前述 $L$、$L_{\mathrm{common}}$ 公式代入单轮成本；求和与闭式等价。

**（2）策略 B（与实现一致的逐轮模拟）**

对第 $n$ 轮，由程序维护 $(L_n,\, C^{\mathrm{common}}_n)$ 并计

$$
C_B^{\mathrm{单轮}}(n) = C_{\mathrm{in}}(L_n, C^{\mathrm{common}}_n) + 10^{-6} A\, c_{\mathrm{out}}.
$$

`simulateB` 在 $n=1\ldots N_{\max}$ 上依 `summaryRound`、`bMode`、$H$、$K$ 更新 `len`、`common`、`rawCount` 等状态，**无需再手写** $S_{\max}$ 的闭式，即可得到与页面一致的**精确数值** $C_B$。

**（3）A 与 B 的累计对齐（页面中的「前 $K$ 轮一致」）**

在首次摘要前（$n \le K$），工具令策略 B 与 A **相同**（`diffSeries`：$n\le K$ 时 B 的累计 = A 的累计）；自 $K+1$ 轮起在 A 的「第 $K$ 轮止」累计上叠加 B 的增量：

$$
C_B(n) = C_A(K) + \sum_{i=K+1}^{n} C_B^{\mathrm{单轮}}(i) \qquad (n > K),
$$

前 $K$ 轮 $C_B(n)=C_A(n)$。这与「$K$ 前未引入摘要差分」的建模一致。

### 4.2 摘要长度成本阈值 $S_{\max}^{(1)}(N)$、$S_{\max}^{(2)}(N)$

**定义：** 给定 $N>20$ 及其它参数，使策略 B1（或 B2）**累计总成本不高于**策略 A 的最大摘要 token 上界，记

$$
S_{\max}^{(1)}(N) = \sup \left\{ S \ge 0 \mathrel{:} \mathcal{C}_{B1}(N, S) \le \mathcal{C}_A(N) \right\},
$$

B2 同理得 $S_{\max}^{(2)}(N)$。下标表示依赖 $N$ 与所有价格、$P$、$A$、$K$、$H$。

**求法说明：** 在「B1 全 $c_{\mathrm{in}}$」的**线性**近似下，$\mathcal{C}_{B1}$ 对 $S$ 常为**仿射**（与摘要出现次数、每轮有效输入长度成比例），而 $\mathcal{C}_A$ 与 $S$ 无关。不等式 $\mathcal{C}_{B1}(N,S) \le \mathcal{C}_A(N)$ 可化为

$$
S \le S_{\ast}(N) = \text{由一次函数解出的显式上界，}
$$

在一段 $N$ 内为单一线性不等式；当模型分段（多段窗口、多段裁剪）时，$S_{\max}$ 为各段上界的**下确界**之 $\min$。

对 **B2**，因缓存命中使 $\mathcal{C}_{B2}$ 对 $S$ **非处处仿射**（有二次项的交叉与分段），**闭式** $S_{\max}^{(2)}$ 更宜写为

$$
S_{\max}^{(2)}(N) = \max \left\{ S \mathrel{:} \mathcal{C}_{B2}(N,S) - \mathcal{C}_A(N) \le 0 \right\},
$$

在固定其它参数时通过**一维单调**搜索或分段解析联立求解；**本页工具**则直接以数值差分 `d = B - A` 回答「是否不劣于 A」，见第6 节。

> **工程注：** 若需论文级闭式，建议在「B1=全程无缓存、输入长度为 $P+S+hA$ 」的稳态近似下展开 $\sum$ 后解 $S$；B2 则给出**分段**不等式。仓库实现侧重可运行的逐轮**精确**比较。

### 4.3 压缩比 $S = KA/R$ 与临界轮数

将 $S$ 代换为 $R$ 的函数后，固定 $K$、$A$、$R$ 时，$C_B - C_A$ 随 $N$ 变化。**页面**在区间表与绿色带中给出满足 $n > K$ 且 $B - A \le 0$ 的**整轮区间**（`findIntervals`），并在曲线上用插值估计**首次打平**附近轮数 `firstCross`（线性插值过零点近似）。

**三种压缩比** $R \in \{R_1, R_2, R_3\}$ 下：分别对参数网格计算上述区间，即可回答**在何 $N$ 范围 A 更便宜、自何处起 B1 或 B2 更占优**；临界点之后占优方由该轮起 `d` 的符号（B−A）决定。
